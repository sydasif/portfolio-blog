### VS Code Setup for Network Engineers

Visual Studio Code (VS Code) is one of the most powerful and customizable code editors available. Having the right settings can significantly boost productivity and make coding more enjoyable, especially for network automation tasks and Python development. In this guide, I’ll walk you through my personal VS Code settings, organized into clear categories. You can adapt these settings to fit your workflow and coding style.

#### Workbench Settings

The workbench is the main interface of VS Code, including elements like sidebars, the activity bar, and the settings editor.

```
    // ----------- Workbench Settings -----------  
    "workbench.startupEditor": "none", // Disable the welcome page on startup  
    "workbench.sideBar.location": "right", // Place the sidebar on the right side  
    "workbench.iconTheme": "vscode-great-icons", // Set the workbench icon theme  
    "workbench.settings.editor": "json", // Use JSON format for settings editor  
    "workbench.colorTheme": "One Monokai", // Set the color theme for the workbench  
    "workbench.tree.indent": 14, // Set tree view indentation  
    "workbench.tree.renderIndentGuides": "onHover", // Render indent guides on hover  
    "workbench.editor.editorActionsLocation": "titleBar",
```

* **Startup Editor**: Disables the welcome page when opening VS Code.
* **Sidebar Location**: Moves the sidebar to the right for a cleaner workspace.
* **Icon and Color Theme**: Uses “VSCode Great Icons” and the “One Monokai” color theme for a sleek look.
* **Settings Format**: Opens settings in JSON format for quick editing.
* **Plugin Requirements**: Requires **One Monokai** and **vscode-great-icons** plugin.

#### Editor Settings

These settings define how the code editor behaves and looks.

```
    // ----------- Editor Settings -----------  
    "editor.fontFamily": "JetBrains Mono Light, Fira Code", // Set the font family for the editor  
    "editor.fontLigatures": true, // Enable font ligatures for the editor  
    "editor.fontSize": 16, // Set the font size for the editor (in pixels)  
    "editor.minimap.enabled": false, // Disable the minimap  
    "editor.unicodeHighlight.nonBasicASCII": false, // Disable breadcrumbs in the editor  
  
    // ----------- Rulers -----------  
    "editor.rulers": [  
        {  
            "column": 110,  
            "color": "#684e4e"  
        }  
    ],
```

* **Font and Ligatures**: Uses **“JetBrains Mono Light”** and **“Fira Code”** with ligatures for better readability.
* **Rulers**: Adds vertical rulers at 88 and 120 characters to maintain code width limits.
* **Minimap**: Disables the minimap for a cleaner editor view.
* **JetBrains Font**: Download [here](https://www.jetbrains.com/lp/mono/).

#### Python-Specific Settings

These settings ensure efficient formatting and code actions for Python, which is essential for network automation tasks.

```
    // ----------- Python-Specific Settings -----------  
    "[python]": {  
        "editor.formatOnSave": true, // Automatically format Python files on save  
        "editor.defaultFormatter": "ms-python.black-formatter", // Use Black as the default formatter  
        "editor.codeActionsOnSave": {  
            "source.fixAll": "explicit" // Apply all explicit fixes on save  
        },  
        "editor.rulers": [  
            {  
                "column": 72,  
                "color": "#684e4e"  
            },  
            {  
                "column": 79,  
                "color": "#4b5c59"  
            }  
        ]  
    },
```

* **Format on Save**: Automatically formats Python files when you save them.
* **Default Formatter**: Uses the **Black** formatter for consistent code style.
* **Code Actions**: Fixes any issues when saving the file.
* **Required Plugins**: **Ruff** Python linter and **Black** formatter.

#### Terminal Settings

Customizing the integrated terminal for better efficiency, is useful for running Ansible playbooks or Python scripts.

```
"terminal.integrated.cursorBlinking": true,  
"terminal.integrated.cursorStyle": "underline",  
"terminal.integrated.fontFamily": "FiraCode Nerd Font Mono",  
"terminal.integrated.fontSize": 14,  
"terminal.integrated.defaultProfile.linux": "zsh",  
"terminal.integrated.defaultProfile.windows": "PowerShell"
```

* **Cursor Settings**: Blinking underline cursor for easy tracking.
* **Font**: Uses **“FiraCode Nerd Font Mono”** for terminal font.
* **Shell Profiles**: Sets `zsh` for Linux and `PowerShell` for Windows as the default terminal.

#### Git and Source Control Settings

Optimizing Git workflows within VS Code for network automation projects.

```
"scm.defaultViewMode": "tree",  
"git.confirmSync": false,  
"git.autofetch": true
```

* **View Mode**: Displays Git repositories in a tree view.
* **Sync Confirmation**: Disables sync confirmation for faster updates.
* **Auto Fetch**: Automatically fetches changes from remote repositories.

#### Debug Settings

```
"debug.terminal.clearBeforeReusing": true
```

* **Clear Terminal**: Clears the terminal before each new debug session for clarity.

#### Explorer Settings

```
"explorer.confirmDragAndDrop": false
```

* **Drag-and-Drop**: Disables confirmation dialogs when dragging and dropping files.

#### Indent Rainbow Settings

Colorful indentation guides to enhance code readability.

```
"indentRainbow.indicatorStyle": "light",  
"indentRainbow.lightIndicatorStyleLineWidth": 1,  
"indentRainbow.colors": [  
    "rgba(255,255,64,0.3)",  
    "rgba(127,255,127,0.3)",  
    "rgba(255,127,255,0.3)",  
    "rgba(79,236,236,0.3)"  
]
```

* **Indicator Style**: Uses a 1-pixel-wide line for indent guides.
* **Colors**: Soft, distinct colors for better visualization of indentation levels.
* **Required Plugin**: indent-rainbow.

### VSCode Short Keys:

#### File Operations

* **Find and Open File in Project**: `Ctrl + P`
* **Open Folder or File**: `Ctrl + O`
* **Open Recent Folder**: `Ctrl + R`
* **Open Recent Closed File**: `Ctrl + E`
* **Switch Open Tabs**: `Ctrl + Tab`
* **Close Tabs**: `Ctrl + W`

#### Explorer Navigation

* **Open/Close File Explorer**: `Ctrl + B`
* **Focus File Explorer**: `Ctrl + Shift + E`

#### Editor Navigation

* **Go to Line Number**: `Ctrl + G`
* **Go to Start of Line**: `Home`
* **Go to End of Line**: `End`
* **Go to Start of File**: `Ctrl + Home`
* **Go to End of File**: `Ctrl + End`
* **Focus Editor**: `Ctrl + 1`

#### Text Editing

* **Multiple Selection**: `Alt + Click`
* **Next Match and Select**: `Ctrl + D`
* **Duplicate Line**: `Alt + Shift + Down`
* **Move Line**: `Alt + Down`
* **Delete Current Line**: `Ctrl + Shift + K`
* **Add Tab**: `Ctrl + ']'`
* **Remove Tab**: `Ctrl + '['`
* **Comment Line**: `Ctrl + /`
* **Format Code**: `Ctrl + Alt + L`

#### Terminal

* **Select Terminal**: `Ctrl + Back-tic`

#### Git

* **Open Git**: `Ctrl + Shift + G`

#### GitHub copilot

* **Open GitHub Copilot**: `Ctrl + Alt + B`
