### **How to Configure and Troubleshoot IPv4 on Cisco Routers — A Hands-On Lab**

Configuring and troubleshooting IPv4 addresses is a fundamental task for any network engineer. Whether preparing for a certification exam or working in a live network environment, mastering IP addressing on Cisco routers is essential. In this lab, we’ll guide you through configuring, verifying, and troubleshooting IPv4 addresses on Cisco routers. You’ll learn how to configure hostnames, interfaces, and loopback interfaces, and use key show commands to confirm that everything is functioning as expected.

#### **Lab Topology Overview**

This lab involves two routers: **\*\*R1\*\*** and **\*\*R2\*\***. Your task will be configuring the hostnames, assigning IP addresses to interfaces, and ensuring proper connectivity between both routers.

![fig. 1](https://cdn-images-1.medium.com/max/800/1*1Fo1OR6w0b83yDwE_zpLvw.png)

#### **Task 1: Configuring Hostnames**

Setting up hostnames on routers is one of the first steps toward identifying devices in your network. Descriptive hostnames simplify troubleshooting and documentation.

Here’s a step-by-step guide for configuring hostnames on R1 and R2:

1. Access the router in **privileged EXEC mode**:

```
Router> enable
```

2. Enter **global configuration mode**:

```
Router# configure terminal
```

3. Configure the hostname for R1:

```
Router(config)# hostname R1
```

4. Similarly, for R2:

```
Router(config)# hostname R2
```

**Explanation:** Unique hostnames help network engineers easily identify routers during troubleshooting and when executing configuration commands.

**Task 2: Configuring Interfaces and Loopback**

1. Configure the **\*\*FastEthernet 0/0\*\*** interface on R1:

```
R1(config)# interface fastEthernet 0/0  
R1(config-if)# ip address 192.168.1.1 255.255.255.0  
R1(config-if)# no shutdown
```

2. Similarly, configure the FastEthernet 0/0 interface on R2:

```
R2(config)# interface fastEthernet 0/0  
R2(config-if)# ip address 192.168.1.2 255.255.255.0  
R2(config-if)# no shutdown
```

**Configuring Loopback Interfaces**

Loopback interfaces are virtual interfaces used for testing and administrative purposes. Since they are always up, they are useful for routing and network management.

1. Configure a Loopback interface on R1:

```
R1(config)# interface loopback 0  
R1(config-if)# ip address 10.0.0.1 255.255.255.0
```

2. Configure a Loopback interface on R2:

```
R2(config)# interface loopback 0  
R2(config-if)# ip address 10.0.0.2 255.255.255.0
```

**Why Loopbacks Matter:** Loopback interfaces are essential for reliable testing, routing, and network monitoring, as they are always active and can be used as the router’s IP address for testing protocols like OSPF or BGP.

**Task 3: Verifying and Troubleshooting IPv4 Configuration**

Once the configurations are complete, it’s essential to verify them using key show commands. This helps to ensure that everything is set up correctly and that there are no issues with the interfaces.

**1. Viewing Configured IP Addresses**

The `show ip interface brief` command offers a quick summary of all interfaces, their IP addresses, and the status (whether they are up or down).

**Command:**

```
R1# show ip interface brief
```

**Example Output:**

```
Interface       IP-Address   OK?   Method   Status                 Protocol  
FastEthernet0/0 192.168.1.1   YES   manual   up                    up  
FastEthernet0/1 unassigned    YES   unset    administratively down down  
Loopback0       10.0.0.1      YES   manual   up                    up
```

**Explanation:** This output shows that FastEthernet0/0 and Loopback0 on R1 are configured and operational. If an interface is listed as “administratively down,” it may need to be enabled with the `no shutdown` command.

**2. Checking Interface Status**

You can further investigate an interface’s status and configuration using the `show interfaces` command.

**Command:**

```
R2# show interfaces fastEthernet 0/0
```

**Example Output:**

```
FastEthernet0/0 is up, line protocol is up  
  Hardware is Gt96k FE, address is c202.169b.0000 (bia c202.169b.0000)  
  Internet address is 192.168.1.2/24  
  MTU 1500 bytes, BW 10000 Kbit/sec, DLY 1000 usec,  
    reliability 255/255, txload 1/255, rxload 1/255  
Encapsulation ARPA, loopback not set  
Half-duplex, 10Mb/s, 100BaseTX/FX  
Last input 00:00:15, output 00:00:06, output hang never  
Input queue: 0/75/0/0 (size/max/drops/flushes); Total output drops: 0  
5 minute input rate 0 bits/sec, 0 packets/sec  
5 minute output rate 0 bits/sec, 0 packets/sec  
    23 packets input, 8096 bytes  
    Received 23 broadcasts, 0 runts, 0 giants, 0 throttles  
    0 input errors, 0 CRC, 0 frame, 0 overrun, 0 ignored  
    0 watchdog  
    203 packets output, 21225 bytes, 0 underruns  
    0 output errors, 0 collisions, 1 interface reset
```

**Explanation of Output:**

**Status:** The interface is up, and the line protocol is also up, indicating that the interface is operational and there are no physical layer issues.

**MAC Address:** The address is c202.169b.0000 (bia c202.169b.0000). This shows the MAC address (Layer 2 hardware address) of the interface, where “bia” stands for Burned-In Address.

**IP Address:** The interface has the IP address 192.168.1.2/24, confirming successful configuration.

**Statistics:** Input/output packet statistics provide insights into the data transmitted and received, along with any errors or drops.

**Performance Metrics:** Metrics such as bandwidth (BW), delay (DLY), and interface reliability offer an overview of the interface’s performance.

**Duplex/Speed:** The interface operates in half-duplex mode, meaning it can either send or receive data, but not at the same time, which may lead to collisions. The connection has a maximum bandwidth of 10Mb/s, while 100BaseTX/FX refers to the Ethernet standards being used: 100BaseTX operates over twisted pair cabling, and 100BaseFX operates over fiber optics.

**Conclusion**

In this lab, you successfully configured IPv4 addresses, verified the status of interfaces, and learned to troubleshoot potential issues on Cisco routers. Being able to use essential show commands allows you to quickly assess the health of your network devices, ensuring that issues are identified and resolved before they cause major disruptions.
